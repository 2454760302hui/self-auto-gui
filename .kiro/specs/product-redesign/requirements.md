# 产品改版需求文档

## 项目概述

完全重构 AutoGLM GUI 产品，打造为商用级别的手机自动化控制平台。改版后的产品应具有专业的界面设计、简化的操作流程、清晰的功能组织，同时保证所有现有功能正常运行。

**改版目标**：从开发工具向商用产品转变，提升用户体验和专业度。

---

## 核心需求

### 1. 功能优先级重新定义

#### 优先级 1（主要功能）
- **手机控制操作**
  - 实时投屏显示
  - 点击、滑动、长按等基础操作
  - 文本输入、按键控制
  - 应用启动、截图等系统操作
  
- **录屏与代码生成**
  - 录制用户操作序列
  - 生成 Pytest 自动化脚本
  - 生成 YAML 配置文件
  - 代码预览、复制、执行

#### 优先级 2（辅助功能）
- **AI 助手**
  - 移至底部或侧边栏
  - 作为辅助功能，非主流程
  - 后续优化性能后再提升优先级

#### 优先级 3（未来功能）
- 用户认证与权限管理（暂不实现）
- 高级分析与报告（后续迭代）

### 2. 界面与交互设计要求

#### 2.1 整体设计原则
- **专业性**：采用企业级设计语言，体现商用产品气质
- **简洁性**：减少视觉噪音，突出核心功能
- **易用性**：降低学习成本，操作流程直观
- **私有化**：完全自主品牌，移除通用设计元素

#### 2.2 色彩系统
- **主色**：深蓝色系（#1e3a8a 或类似）- 专业、可信
- **辅助色**：青色/绿色 - 操作反馈、成功状态
- **背景**：深色主题优先（#0f172a）- 减少眼睛疲劳
- **文本**：高对比度，确保可读性

#### 2.3 布局结构
```
┌─────────────────────────────────────────────┐
│  顶部导航栏（简洁，仅显示关键信息）          │
├──────────────────┬──────────────────────────┤
│                  │                          │
│  左侧面板        │  中央工作区              │
│  - 设备列表      │  - 投屏显示              │
│  - 快速操作      │  - 录屏预览              │
│  - 功能导航      │  - 代码编辑              │
│                  │                          │
├──────────────────┴──────────────────────────┤
│  底部面板（AI 助手、日志等）                 │
└─────────────────────────────────────────────┘
```

### 3. 功能模块设计

#### 3.1 设备管理模块
**位置**：左侧面板顶部
**功能**：
- 设备列表展示（在线/离线状态）
- 快速设备切换
- 设备信息查看（型号、系统版本等）
- 设备连接状态实时更新

**交互**：
- 单击选择设备
- 右键菜单：刷新、断开连接等

#### 3.2 控制操作模块
**位置**：左侧面板中部
**功能**：
- 基础操作按钮（点击、滑动、长按等）
- 快捷键盘（输入文本）
- 系统操作（返回、主页、截图等）
- 应用启动快捷方式

**交互**：
- 按钮式操作，点击即执行
- 支持快捷键
- 操作历史记录

#### 3.3 投屏与录屏模块
**位置**：中央工作区
**功能**：
- 实时投屏显示（左侧）
- 录屏控制面板（右侧）
- 分屏展示，互不干扰

**交互**：
- 投屏区域支持点击操作
- 录屏面板显示操作序列
- 实时预览录制内容

#### 3.4 代码生成与执行模块
**位置**：中央工作区下方或右侧
**功能**：
- 代码预览（Pytest/YAML）
- 代码复制到剪贴板
- 代码执行与调试
- 执行结果展示

**交互**：
- 标签页切换（Pytest/YAML）
- 一键复制、执行
- 结果实时反馈

#### 3.5 AI 助手模块
**位置**：底部面板或可折叠侧边栏
**功能**：
- 自然语言任务描述
- AI 执行反馈
- 执行历史查看

**交互**：
- 最小化/展开
- 不影响主要操作流程

## 用户流程

### 流程 1：基础手机控制
```
1. 选择设备 → 2. 投屏显示 → 3. 点击/滑动操作 → 4. 实时反馈
```

### 流程 2：录屏与代码生成
```
1. 选择设备 → 2. 开始录屏 → 3. 执行操作序列 → 4. 停止录屏 
→ 5. 生成代码 → 6. 预览/复制/执行
```

### 流程 3：AI 辅助操作（可选）
```
1. 打开 AI 助手 → 2. 输入任务描述 → 3. AI 执行 → 4. 查看结果
```

## 技术需求

### 4.1 功能保证
- ✅ 所有现有功能必须正常运行
- ✅ 投屏功能稳定可靠
- ✅ 录屏与代码生成准确无误
- ✅ 操作反馈实时响应

### 4.2 性能要求
- 投屏延迟 < 500ms
- 操作响应 < 200ms
- 代码生成 < 2s
- 页面加载 < 3s

### 4.3 兼容性
- 支持 Android 设备
- 支持 HarmonyOS 设备
- 支持多设备同时连接
